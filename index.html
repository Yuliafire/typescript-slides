TypeScript: Unlocking Type Safety

What is TypeScript?

TypeScript, a superset of JavaScript developed by Microsoft, is a statically typed language that compiles to plain JavaScript.

By introducing static typing, TypeScript enhances the development experience by detecting errors early and providing improved tooling support.

Why Use TypeScript?

Part 1

  • Catch errors early: TypeScript's typing helps spot mistakes, making code reliable.
  • Boost productivity: Features like auto-completion make coding faster.
  • Write scalable code: Supports OOP for clean and organized applications.

Why Use TypeScript?

Part 2

  • TypeScript provides support for the latest ECMAScript features, enabling modern JavaScript syntax.
  • TypeScript is fully compatible with existing JavaScript codebases, allowing seamless integration.

Understanding TypeScript's Type System

TypeScript extends JavaScript by adding a rich type system. While JavaScript uses dynamic typing, TypeScript introduces enums, tuples, and union/intersection types for safer, maintainable code.

1. Primitive Types

  • boolean
  • number
  • string
  • bigint
  • symbol
  • undefined
  • null

2. Special Types

  • any
  • void
  • never
  • unknown

any

The any type disables type checking, allowing any value. Avoid it to maintain TypeScript's type safety.

                    let value: any;
value = "Hello"; // Valid
value = 42;      // Valid
value.toUpperCase(); // No compile-time error, but runtime error if not string

unknown

The unknown type is a safer alternative to any, requiring type checking.

                    let value: unknown = "Hello";
if (typeof value === "string") {
    console.log(value.toUpperCase()); // Safe
}

never

The never type represents values that never occur, like error-throwing functions.

                    function error(message: string): never {
    throw new Error(message);
}

void

The void type represents functions that return nothing.

                    function logMessage(message: string): void {
    console.log(message);
}

3. Comparing Interfaces & Type Aliases

interface and type define data structures but have distinct features.

Explore their differences in extending, merging, and type representation.

Interfaces Can Be Extended

Interfaces use extends to build on existing ones.

                    interface Person {
    name: string;
    age: number;
}
interface Employee extends Person {
    jobTitle: string;
}
const employee: Employee = {
    name: "Alice",
    age: 30,
    jobTitle: "Software Engineer"
};

Interfaces Support Declaration Merging

Interfaces can be reopened to add properties.

                    interface Person {
    name: string;
}
// Declaration merging
interface Person {
    middleName?: string;
}
const person: Person = {
    name: "Alice",
    middleName: "Marie"
};

Type Aliases Can Represent Primitives, Unions, Tuples

Type aliases are flexible for primitives, unions, and tuples.

                    // Union type
type StringOrNumber = string | number;
const value: StringOrNumber = 42;
// Tuple type
type Point = [number, number];
const point: Point = [10, 20];

4. Composite Types

Composite types group multiple values.

Includes ARRAYS, TUPLES, UNION, and INTERSECTION.

Array

An array holds elements of the same type.

                    let numbers: number[] = [1, 2, 3, 4];
numbers.push(5); // Valid
numbers.push("hello"); // Error: Argument of type 'string' is not assignable

Tuple

A tuple is a fixed-length array with different types.

                    let person: [string, number] = ["Alice", 25];
// person = [25, "Alice"]; // Error: Type 'number' is not assignable to type 'string'

Union Types

A union type allows a value to be one of several types.

                    type StringOrNumber = string | number;
let value: StringOrNumber;
value = "Hello"; // Valid
value = 42;      // Valid
value = true;    // Invalid

Intersection Types

An intersection type combines multiple types.

                    type Person = { name: string };
type Employee = { jobTitle: string };
type PersonEmployee = Person & Employee;
const employee: PersonEmployee = {
    name: "Alice",
    jobTitle: "Developer"
};

5. Enum Types

Enums define named constants.

                    enum Color {
    Red,
    Green,
    Blue
}
let favoriteColor: Color = Color.Green;
console.log(favoriteColor); // Output: 1

String Literal

Literal types represent specific values, often used with unions.

  • String Literal: Restricts to a specific string.
    •                         let status: "success" | "error" = "success";

Numeric and Boolean Literals

Literal types represent specific values, often used with unions.

  • Numeric Literal: Restricts to a specific number.
    •                         let diceRoll: 1 | 2 | 3 | 4 | 5 | 6 = 3;
  • Boolean Literal: Restricts to true or false.
    •                         let isActive: true = true;

6. Generic Types

Generics create reusable code with type placeholders.

  • Generic Types: Functions or classes that work with any type.
    •                         function identity(value: T): T {
    return value;
}
let str = identity("Hello"); // T is string
let num = identity(42); // T is number

Generic Constraints

Generics create reusable code with type placeholders.

  • Generic Constraints: Restrict types to meet requirements.
    •                         function getLength(item: T): number {
    return item.length;
}
let len1 = getLength("Hello"); // Works
let len2 = getLength([1, 2, 3]); // Works

7. Type Assertions

Advanced types enable complex type manipulations.

  • Type Assertions: Specify a value’s type when known.
    •                         let value: any = "Hello";
let strLength: number = (value as string).length;

Non-null Assertion

Advanced types enable complex type manipulations.

  • Non-null Assertion: Asserts non-null/undefined.
    •                         let element: HTMLElement | null = document.getElementById("app");
let definitelyElement: HTMLElement = element!;

8. Utility Types

Utility types transform or manipulate other types for complex definitions.

Partial

  • Partial: Makes all properties optional.
    •                         interface User {
    id: number;
    name: string;
}
let partialUser: Partial = { name: "John" };

Readonly

  • Readonly: Makes all properties read-only.
    •                         interface User {
    id: number;
    name: string;
}
let readonlyUser: Readonly = { id: 1, name: "John" };
// readonlyUser.name = "Doe"; // Error

Pick

  • Pick: Selects specific properties.
    •                         interface User {
    id: number;
    name: string;
    email: string;
}
let pickedUser: Pick = { id: 1, name: "John" };

Omit

  • Omit: Excludes specific properties.
    •                         interface User {
    id: number;
    name: string;
    email: string;
}
let omittedUser: Omit = { id: 1, name: "John" };

  • Enhanced Tooling: Better autocompletion and debugging.
